# 4. Workflow Integration over Standalone Tools

We prioritize embedding into workflows users already perform over creating
new ones.
Users do not adopt tools that require behavior change.

- _Decision check:_ Name the specific existing workflow this fits into. If the answer is "a new one," reconsider.
- _Status:_ enforceable.

## Open questions

- Are there cases where creating a new workflow is genuinely necessary
  (e.g., when no existing workflow exists)? If so, how do we recognize them
  vs. wishful thinking?
- Does "workflow" need a tighter definition? Today it could mean
  "what users do in a day" or "a specific process they already follow."
